# üü© **2. What is Auto-Configuration? (Simple Explanation)**

Spring Boot automatically configures your project based on the dependencies added in **pom.xml**.

### Example:

If you add:

```xml
spring-boot-starter-web
```

Spring Boot automatically configures:

* Tomcat Server
* DispatcherServlet
* JSON converter
* Error handling
* REST APIs setup

You didn‚Äôt write a single line ‚Üí Spring Boot does it automatically.

# üü• **3. What is @SpringBootApplication?**

This annotation is used on the main class.

It contains:

### ‚úî @Configuration

Creates beans.

### ‚úî @EnableAutoConfiguration

Enables auto configuration.

### ‚úî @ComponentScan

Scans project to find:

* @Controller
* @Service
* @Repository
* @Component

This is why Spring Boot runs with zero configuration.

# üüß **4. What are Spring Boot Starters? (Very Important)**

Starters are **ready-made dependency packs**.

### Example:

To create REST APIs ‚Üí we need web libraries.
So we use:

```xml
spring-boot-starter-web
```

This includes:

* REST support
* Tomcat
* JSON
* Basic logging

### Common Starters:

| Starter                        | Purpose                        |
| ------------------------------ | ------------------------------ |
| spring-boot-starter-web        | Build REST APIs                |
| spring-boot-starter-data-jpa   | Database ORM                   |
| spring-boot-starter-security   | Authentication & Authorization |
| spring-boot-starter-validation | Input validations              |
| spring-boot-starter-thymeleaf  | HTML UI                        |

Starters remove 80% of manual configuration.

# üü´ **7. application.yml (Beginner Friendly Explanation)**

# üüß **8. Creating Custom Properties**

Add in application.properties:

```properties
app.title=Spring Boot Training
app.version=1.0
```

Use @Value to access:

```java
@Value("${app.title}")
private String title;
```

Show in API:

```java
@GetMapping("/info")
public String info() {
    return title;
}
```

# üü¶ **9. Profiles (dev, test, prod) ‚Äì Beginner Level**

Profiles allow different settings for different environments.

### **Step 1: Create files**

```
application-dev.properties
application-prod.properties
```

### **dev file:**

```properties
server.port=8081
message=Running in DEV mode
```

### **prod file:**

```properties
server.port=8082
message=Running in PROD mode
```

---

### **Step 2: Activate a profile**

In `application.properties`:

```properties
spring.profiles.active=dev
```

Now your app uses:

```
application-dev.properties
```

Change dev ‚Üí prod easily.

---

### **Step 3: Read profile value**

```java
@Value("${message}")
private String msg;
```

```java
@GetMapping("/profile")
public String getProfile() {
    return msg;
}
```

# üü• **10. CommandLineRunner (Beginner Explanation)**

CommandLineRunner runs code **after** Spring Boot starts.

### Example:

```java
@Component
public class StartupClass implements CommandLineRunner {

    @Override
    public void run(String... args) {
        System.out.println("Application started successfully!");
    }
}
```

Use cases:

* Print messages
* Load sample data
* Check DB connection
