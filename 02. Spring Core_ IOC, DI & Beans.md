# ğŸŸ§ **3. What is IOC (Inversion of Control)?**

### **Simple Meaning:**

Normally, programmer creates objects.
But in Spring, **Spring Container creates objects**.

Control goes from programmer â†’ Spring.

### Without IOC:

* Spring manages their life

# ğŸŸ¦ **5. What is a Spring Bean?**

Any object created and managed by Spring Container is called a **Bean**.

Spring creates beans for classes annotated with:

* @Component
* @Service
* @Repository
* @Controller

Example:

```java

}
```

Spring will automatically create object of StudentService.

# ğŸŸ§ **7. Constructor Injection (Recommended)**

### Why?

* Best practice
* Easy to test
* Avoids null values

### Example:

#### **Repository**

```java
@Repository
public class StudentRepository {

    public String getStudent() {
        return "Vijay";
    }
}
```

#### **Service**

```java
@Service
public class StudentService {

    private final StudentRepository repo;

    public StudentService(StudentRepository repo) {  // DI here
        this.repo = repo;
    }

    public String getStudentName() {
        return repo.getStudent();
    }
}
```

#### **Controller**

```java
@RestController
public class StudentController {

    private final StudentService service;

    public StudentController(StudentService service) {  // DI here
        this.service = service;
    }

    @GetMapping("/student")
    public String getStudent() {
        return service.getStudentName();
    }
}
```

### Flow:

```
Controller â†’ Service â†’ Repository
(All are injected automatically)
```
